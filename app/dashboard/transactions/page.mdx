# แท็บธุรกรรม (Transactions)

แท็บนี้มุ่งเน้นไปที่ข้อมูลรายละเอียดและแนวโน้มของธุรกรรมทั้งหมด

## การ์ดข้อมูลสำคัญ (KPI Cards)

1.  **Today (วันนี้):**
    *   **แสดงอะไร:** จำนวนธุรกรรมทั้งหมดที่เกิดขึ้นในวันนี้ (เหมือนกับการ์ด "Transactions Today" ในแท็บ Overview)
    *   **คำอธิบายเพิ่มเติม (ด้านล่างตัวเลขหลัก):**
        *   ไอคอน TrendUp สีเขียว (TrendingUp) พร้อมตัวเลข "X in": แสดงจำนวนธุรกรรมขาเข้าในวันนี้
        *   ไอคอน TrendDown สีแดง (TrendingDown) พร้อมตัวเลข "Y out": แสดงจำนวนธุรกรรมขาออกในวันนี้
    *   **ไอคอน:** รูปปฏิทิน (Calendar)
    *   **วิธีใช้:** ภาพรวมกิจกรรมธุรกรรมรายวัน พร้อมแยกจำนวนเข้า-ออกอย่างชัดเจน

2.  **This Week (สัปดาห์นี้):**
    *   **แสดงอะไร:** จำนวนธุรกรรมทั้งหมดที่เกิดขึ้นในสัปดาห์นี้ (นับถึงปัจจุบัน)
    *   **คำอธิบายเพิ่มเติม:** "X% today" หมายถึง สัดส่วนของธุรกรรมที่เกิดขึ้นในวันนี้เมื่อเทียบกับยอดรวมทั้งสัปดาห์
    *   **ไอคอน:** รูปแผนภูมิแท่ง (BarChart3)
    *   **วิธีใช้:** ติดตามปริมาณธุรกรรมสะสมในสัปดาห์ปัจจุบัน

3.  **This Month (เดือนนี้):**
    *   **แสดงอะไร:** จำนวนธุรกรรมทั้งหมดที่เกิดขึ้นในเดือนนี้ (นับถึงปัจจุบัน)
    *   **คำอธิบายเพิ่มเติม:** "X% this week" หมายถึง สัดส่วนของธุรกรรมที่เกิดขึ้นในสัปดาห์นี้เมื่อเทียบกับยอดรวมทั้งเดือน
    *   **ไอคอน:** รูปแผนภูมิแท่ง (BarChart3)
    *   **วิธีใช้:** ติดตามปริมาณธุรกรรมสะสมในเดือนปัจจุบัน

4.  **Pending (รอดำเนินการ):**
    *   **แสดงอะไร:** จำนวนธุรกรรมทั้งหมดที่ยังอยู่ในสถานะ "รอดำเนินการ" (เหมือนกับการ์ด "Pending Transactions" ในแท็บ Overview)
    *   **คำอธิบายเพิ่มเติม:** "X unprocessed groups" หมายถึง มีกลุ่มธุรกรรมจำนวน X กลุ่มที่ยังไม่ถูกประมวลผล
    *   **ไอคอน:** รูปวงกลมมีเครื่องหมายตกใจสีเหลือง (AlertCircle)
    *   **วิธีใช้:** ใช้เพื่อตรวจสอบจำนวนงานที่ค้างอยู่ และจำเป็นต้องดำเนินการให้เสร็จสิ้น

## แผนภูมิ (Charts)

1.  **Transaction Status (สถานะธุรกรรม):**
    *   **ประเภทแผนภูมิ:** แผนภูมิวงกลม (Pie Chart)
    *   **แสดงอะไร:** สัดส่วนของธุรกรรมตามสถานะต่างๆ เช่น รอดำเนินการ (PENDING), เสร็จสมบูรณ์ (COMPLETED), ล้มเหลว (FAILED)
    *   **คำอธิบาย:** "Distribution by status" (การกระจายตามสถานะ)
    *   **วิธีใช้:**
        *   ดูขนาดของแต่ละชิ้นส่วนเพื่อเข้าใจว่าธุรกรรมส่วนใหญ่อยู่ในสถานะใด
        *   ให้ความสนใจกับสถานะ "FAILED" หรือ "PENDING" จำนวนมากเป็นพิเศษ
        *   เลื่อนเมาส์ไปบนแต่ละชิ้นส่วนเพื่อดูชื่อสถานะและเปอร์เซ็นต์
        *   คำอธิบาย (Legend) ด้านล่างจะบอกว่าสีไหนแทนสถานะใด

2.  **Weekly Trend (แนวโน้มรายสัปดาห์):**
    *   **ประเภทแผนภูมิ:** แผนภูมิเส้น (Line Chart)
    *   **แสดงอะไร:** แนวโน้มจำนวนธุรกรรมรวมที่เกิดขึ้นในแต่ละสัปดาห์ย้อนหลังไป แกน X (แนวนอน) คือวันที่เริ่มต้นสัปดาห์ และแกน Y (แนวตั้ง) คือจำนวนธุรกรรมรวมของสัปดาห์นั้น
    *   **คำอธิบาย:** "Transaction volume by week" (ปริมาณธุรกรรมตามสัปดาห์)
    *   **วิธีใช้:**
        *   สังเกตแนวโน้มของเส้นกราฟเพื่อดูว่าปริมาณธุรกรรมรายสัปดาห์เพิ่มขึ้น ลดลง หรือคงที่
        *   เปรียบเทียบจุดบนเส้นกราฟเพื่อดูความแตกต่างของปริมาณธุรกรรมระหว่างสัปดาห์
        *   เลื่อนเมาส์ไปบนจุดต่างๆ (dot) บนเส้นกราฟเพื่อดูจำนวนธุรกรรมที่แน่นอนของสัปดาห์นั้น

## ตารางสรุปธุรกรรม (Transaction Summary Tables)

1.  **Top Inflow by Quantity (ยอดนำเข้าสูงสุดตามปริมาณ):**
    *   **แสดงอะไร:** รายการสินค้า 5 อันดับแรกที่มีการนำเข้า (ธุรกรรมประเภท IN) มากที่สุดตามปริมาณ
    *   **คำอธิบาย:** "Most received items" (รายการสินค้าที่ได้รับบ่อยที่สุด/มากที่สุด)
    *   **คอลัมน์ในตาราง:**
        *   **Stock (สินค้า):** ชื่อสินค้า
        *   **Quantity (จำนวน):** ปริมาณรวมที่นำเข้าของสินค้านั้น
    *   **วิธีใช้:** ระบุสินค้าที่มีการรับเข้าคลังบ่อยหรือปริมาณมาก เพื่อวางแผนการจัดเก็บและการจัดการ

2.  **Top Outflow by Quantity (ยอดนำออกสูงสุดตามปริมาณ):**
    *   **แสดงอะไร:** รายการสินค้า 5 อันดับแรกที่มีการนำออก (ธุรกรรมประเภท OUT) มากที่สุดตามปริมาณ
    *   **คำอธิบาย:** "Most distributed items" (รายการสินค้าที่มีการกระจายออกไปบ่อยที่สุด/มากที่สุด)
    *   **คอลัมน์ในตาราง:**
        *   **Stock (สินค้า):** ชื่อสินค้า
        *   **Quantity (จำนวน):** ปริมาณรวมที่นำออกของสินค้านั้น
    *   **วิธีใช้:** ระบุสินค้าขายดีหรือสินค้าที่มีการเบิกใช้บ่อย เพื่อวางแผนการเติมสต็อกให้เพียงพอ

## แผนภูมิธุรกรรมต่อผู้ใช้ (Transactions Per User Chart)

*   **ประเภทแผนภูมิ:** แผนภูมิแท่งแนวนอน (Horizontal Bar Chart)
*   **แสดงอะไร:** จำนวนธุรกรรมทั้งหมดที่ดำเนินการโดยผู้ใช้แต่ละคน แกน Y (แนวตั้ง) คืออีเมลผู้ใช้ (อาจแสดงย่อถ้าชื่อยาว) และแกน X (แนวนอน) คือจำนวนธุรกรรม
*   **คำอธิบาย:** "User activity breakdown" (รายละเอียดกิจกรรมของผู้ใช้)
*   **วิธีใช้:**
    *   เปรียบเทียบความยาวของแท่งกราฟเพื่อดูว่าผู้ใช้คนใดมีส่วนร่วมในการทำธุรกรรมมากที่สุด
    *   อาจใช้ในการประเมินภาระงานหรือความถี่ในการใช้งานระบบของผู้ใช้แต่ละคน
    *   เลื่อนเมาส์ไปบนแท่งกราฟเพื่อดูจำนวนธุรกรรมที่แน่นอนของผู้ใช้นั้น

## ตารางกลุ่มธุรกรรมที่รอดำเนินการ (Pending Transaction Groups Table)

*   **แสดงอะไร:** รายการกลุ่มธุรกรรมที่ยังคงรอดำเนินการหรือยังไม่ถูกประมวลผล
*   **คำอธิบาย:** "Groups awaiting processing" (กลุ่มที่รอการประมวลผล)
*   **ไอคอนหัวข้อ:** รูปวงกลมมีเครื่องหมายตกใจสีเหลือง (AlertCircle)
*   **คอลัมน์ในตาราง:**
    *   **Group ID (รหัสกลุ่ม):** หมายเลขอ้างอิงของกลุ่มธุรกรรม
    *   **Name (ชื่อ):** ชื่อของกลุ่มธุรกรรม
    *   **User (ผู้ใช้):** อีเมลของผู้ใช้ที่สร้างกลุ่มธุรกรรมนั้น
    *   **Created (สร้างเมื่อ):** วันที่และเวลาที่กลุ่มธุรกรรมถูกสร้างขึ้น แสดงในรูปแบบ "X [หน่วยเวลา] ago"
*   **วิธีใช้:**
    *   ตรวจสอบกลุ่มงานที่ค้างอยู่และจำเป็นต้องดำเนินการต่อ
    *   ใช้เพื่อติดตามความคืบหน้าของกลุ่มธุรกรรมที่สำคัญ

## เคล็ดลับและเทคนิคเพิ่มเติม

*   **รีเฟรชข้อมูล:** โดยทั่วไปข้อมูลในแดชบอร์ดจะถูกโหลดเมื่อคุณเปิดหน้า หากต้องการข้อมูลล่าสุด อาจจำเป็นต้องรีเฟรชหน้าเว็บเบราว์เซอร์ของคุณ (กด F5 หรือปุ่มรีเฟรช)
*   **ตรวจสอบเป็นประจำ:** เข้ามาดูแดชบอร์ดเป็นประจำ (เช่น ทุกเช้า) เพื่อติดตามสถานการณ์ล่าสุดและระบุปัญหาที่อาจเกิดขึ้นได้อย่างรวดเร็ว
*   **ใช้ร่วมกับรายงานอื่น:** ข้อมูลในแดชบอร์ดเป็นภาพรวม หากต้องการรายละเอียดลึกๆ ของข้อมูลเฉพาะส่วน อาจต้องไปดูในส่วนรายงานอื่นๆ ของระบบ (ถ้ามี)

## การอ่านข้อมูลและการตีความ

*   **สังเกตแนวโน้ม:** อย่าดูแค่ตัวเลขปัจจุบัน แต่ให้สังเกตแนวโน้มเมื่อเวลาผ่านไป (เช่น ใน Transaction Trend หรือ Weekly Trend) เพื่อเข้าใจทิศทางของธุรกิจ
*   **เปรียบเทียบข้อมูล:** เปรียบเทียบข้อมูลระหว่างช่วงเวลาต่างๆ หรือระหว่างกลุ่มสินค้าต่างๆ เพื่อหาความผิดปกติหรือโอกาส
*   **ระบุสิ่งผิดปกติ (Anomalies):** หากตัวเลขบางอย่างสูงหรือต่ำผิดปกติอย่างมาก (เช่น Pending Transactions สูงปรี๊ด) ควรตรวจสอบหาสาเหตุ
*   **เชื่อมโยงข้อมูล:** ลองเชื่อมโยงข้อมูลจากส่วนต่างๆ เข้าด้วยกัน เช่น หาก "Low Stock Items" มีจำนวนมาก อาจส่งผลต่อ "Total Stock Value" ในอนาคต หรือหาก "Pending Transactions" สูง อาจหมายถึงมีงานค้างที่ต้องรีบจัดการ
