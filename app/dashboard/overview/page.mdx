# แท็บภาพรวม (Overview)

แท็บนี้จะแสดงข้อมูลสรุปที่สำคัญที่สุดจากทั้งส่วนของสต็อกและธุรกรรม ช่วยให้คุณเห็นสถานการณ์ปัจจุบันของระบบได้อย่างรวดเร็ว

## การ์ดข้อมูลสำคัญ (KPI Cards)

ส่วนนี้แสดงข้อมูลตัวชี้วัดหลัก (Key Performance Indicators) ในรูปแบบการ์ดที่อ่านง่าย:

1.  **Total Stock Value (มูลค่าสต็อกทั้งหมด):**
    *   **แสดงอะไร:** มูลค่ารวมของสต็อกสินค้าทั้งหมดที่มีอยู่ในระบบ คิดเป็นสกุลเงินบาท (THB)
    *   **คำอธิบายเพิ่มเติม:** "Across X unique items" หมายถึง มูลค่านี้คำนวณจากสินค้าที่ไม่ซ้ำกันจำนวน X รายการ
    *   **ไอคอน:** รูปถุงเงิน (DollarSign)
    *   **วิธีใช้:** ใช้เพื่อประเมินมูลค่าสินทรัพย์คงคลังทั้งหมดของธุรกิจ

2.  **Total Stock Count (จำนวนสต็อกทั้งหมด):**
    *   **แสดงอะไร:** จำนวนชิ้นของสินค้าทั้งหมดที่มีอยู่ในคลัง
    *   **คำอธิบายเพิ่มเติม:** "Items in inventory" หมายถึง จำนวนสินค้าทั้งหมดในคลัง
    *   **ไอคอน:** รูปกล่องพัสดุ (Package)
    *   **วิธีใช้:** ใช้เพื่อทราบปริมาณสินค้าทั้งหมดที่มี ช่วยในการวางแผนการจัดการพื้นที่คลัง

3.  **Transactions Today (ธุรกรรมวันนี้):**
    *   **แสดงอะไร:** จำนวนธุรกรรมทั้งหมดที่เกิดขึ้นในวันนี้
    *   **คำอธิบายเพิ่มเติม (ด้านล่างตัวเลขหลัก):**
        *   ลูกศรชี้ขึ้นสีเขียว (ArrowUp) พร้อมตัวเลข: แสดงจำนวนธุรกรรมขาเข้า (สินค้าเข้าคลัง) ที่เกิดขึ้นในวันนี้
        *   ลูกศรชี้ลงสีแดง (ArrowDown) พร้อมตัวเลข: แสดงจำนวนธุรกรรมขาออก (สินค้าออกจากคลัง) ที่เกิดขึ้นในวันนี้
    *   **ไอคอน:** รูปตะกร้าสินค้า (ShoppingCart)
    *   **วิธีใช้:** ติดตามกิจกรรมธุรกรรมรายวัน แยกดูจำนวนสินค้าเข้าและออกเพื่อเข้าใจการหมุนเวียนสินค้า

4.  **Pending Transactions (ธุรกรรมที่รอดำเนินการ):**
    *   **แสดงอะไร:** จำนวนธุรกรรมทั้งหมดที่ยังอยู่ในสถานะ "รอดำเนินการ"
    *   **คำอธิบายเพิ่มเติม:** "X unprocessed groups" หมายถึง มีกลุ่มธุรกรรมจำนวน X กลุ่มที่ยังไม่ถูกประมวลผล
    *   **ไอคอน:** รูปนาฬิกา (Clock)
    *   **วิธีใช้:** ใช้เพื่อตรวจสอบจำนวนงานที่ค้างอยู่ และจำเป็นต้องดำเนินการให้เสร็จสิ้น

## แผนภูมิ (Charts)

1.  **Transaction Trend (แนวโน้มธุรกรรม):**
    *   **ประเภทแผนภูมิ:** แผนภูมิพื้นที่ (Area Chart)
    *   **แสดงอะไร:** แสดงแนวโน้มจำนวนธุรกรรมที่เกิดขึ้นในแต่ละวันย้อนหลังไปช่วงระยะเวลาหนึ่ง แกน X (แนวนอน) คือวันที่ และแกน Y (แนวตั้ง) คือจำนวนธุรกรรม
    *   **คำอธิบาย:** "Daily transaction volume over time" (ปริมาณธุรกรรมรายวันในช่วงเวลา)
    *   **วิธีใช้:**
        *   ดูความสูงของพื้นที่ใต้กราฟเพื่อประเมินปริมาณธุรกรรมในแต่ละวัน
        *   สังเกตแนวโน้มโดยรวมว่าเพิ่มขึ้น, ลดลง, หรือคงที่ เพื่อเข้าใจกิจกรรมของระบบ
        *   เลื่อนเมาส์ไปบนกราฟเพื่อดูจำนวนธุรกรรมที่แน่นอนในแต่ละวัน

2.  **Transaction Types (ประเภทธุรกรรม):**
    *   **ประเภทแผนภูมิ:** แผนภูมิวงกลม (Pie Chart)
    *   **แสดงอะไร:** สัดส่วนของประเภทธุรกรรมต่างๆ ที่เกิดขึ้น เช่น สินค้าเข้า (IN), สินค้าออก (OUT), การปรับปรุงสต็อก (ADJUSTMENT) ฯลฯ
    *   **คำอธิบาย:** "Distribution by transaction type" (การกระจายตามประเภทธุรกรรม)
    *   **วิธีใช้:**
        *   ดูขนาดของแต่ละชิ้นส่วนในวงกลมเพื่อเปรียบเทียบสัดส่วนของธุรกรรมแต่ละประเภท
        *   เลื่อนเมาส์ไปบนแต่ละชิ้นส่วนเพื่อดูชื่อประเภทและเปอร์เซ็นต์ที่แน่นอน
        *   คำอธิบาย (Legend) ด้านล่างจะบอกว่าสีไหนแทนประเภทใด

## ตารางธุรกรรมล่าสุด (Recent Transactions Table)

*   **แสดงอะไร:** รายการธุรกรรมล่าสุด 5 รายการที่เกิดขึ้นในระบบ
*   **คำอธิบาย:** "Latest transaction activities" (กิจกรรมธุรกรรมล่าสุด)
*   **คอลัมน์ในตาราง:**
    *   **ID:** หมายเลขอ้างอิงของธุรกรรม
    *   **Stock (สินค้า):** ชื่อของสินค้าที่เกี่ยวข้องกับธุรกรรม
    *   **Type (ประเภท):** ประเภทของธุรกรรม จะแสดงเป็นป้าย (Badge) พร้อมไอคอนและสีที่แตกต่างกัน:
        *   เช่น `IN` (เข้า), `OUT` (ออก), `ADJUSTMENT` (ปรับปรุง), `TRANSFER` (โอนย้าย)
        *   ไอคอนและสีช่วยให้แยกแยะประเภทธุรกรรมได้ง่าย
    *   **Quantity (จำนวน):** จำนวนสินค้าในธุรกรรมนั้น
    *   **Status (สถานะ):** สถานะปัจจุบันของธุรกรรม จะแสดงเป็นป้าย (Badge) พร้อมไอคอนและสี:
        *   เช่น `PENDING` (รอดำเนินการ), `COMPLETED` (เสร็จสมบูรณ์), `FAILED` (ล้มเหลว), `CANCELLED` (ยกเลิก)
    *   **User (ผู้ใช้):** อีเมลของผู้ใช้ที่ทำธุรกรรมนั้น
    *   **Date (วันที่):** วันที่และเวลาที่ธุรกรรมถูกสร้างขึ้น แสดงในรูปแบบ "X [หน่วยเวลา] ago" (เช่น 5 นาทีที่แล้ว, 2 ชั่วโมงที่แล้ว)
*   **วิธีใช้:**
    *   ตรวจสอบกิจกรรมล่าสุดได้อย่างรวดเร็ว
    *   ใช้เป็นจุดเริ่มต้นในการตรวจสอบหากมีปัญหาเกิดขึ้นกับธุรกรรมล่าสุด
